# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability, please report it to us responsibly.

Please do **not** open public GitHub issues for security reports.

## Supported Versions

Only the latest version of this project is currently supported for security updates.

## Scope of Security Reports

We welcome reports related to:
- Authentication issues
- Data exposure or leaks
- Broken access control
- Unsafe storage of sensitive data
- API vulnerabilities
- Dependency-related security risks

Reports that are **out of scope**:
- Feature requests
- Styling or UI bugs
- Non-security crashes
- Issues in third-party libraries we do not control
